# Railway Deployment Guide

## Prerequisites

- Railway account
- Railway CLI installed: `npm i -g @railway/cli`
- GitHub repository

## 1. Create Project from GitHub

```bash
railway login
railway init
# Select "Deploy from GitHub repo"
# Choose your repository
```

## 2. Add Postgres Database

In Railway dashboard:
- Click "New" → "Database" → "Add PostgreSQL"
- Railway auto-generates `DATABASE_URL` and related variables

## 3. Configure Environment Variables

Add these variables in Railway dashboard or via CLI if not automatically added:

```bash
# Server Configuration
railway variables --set PORT=3000
railway variables --set NODE_ENV=production
railway variables --set LOG_LEVEL=info

# Security & CORS
railway variables --set CORS_ORIGIN=https://your-frontend-url.railway.app
railway variables --set QUERY_TIMEOUT=30000
railway variables --set MAX_RESULT_ROWS=1000

# Clerk Authentication
railway variables --set CLERK_PUBLISHABLE_KEY=pk_live_xxxxx
railway variables --set CLERK_SECRET_KEY=sk_live_xxxxx
railway variables --set VITE_CLERK_PUBLISHABLE_KEY=pk_live_xxxxx

# Encryption (generate random 32-byte base64 string)
railway variables --set ENCRYPTION_KEY=your-base64-encoded-key

# Frontend (Vite)
railway variables --set VITE_API_URL=https://your-backend-url.railway.app
railway variables --set VITE_API_TIMEOUT=30000

# Features
railway variables --set ENABLE_QUERY_LOGGING=true
railway variables --set ENABLE_EXPLAIN=true

# Database Pool (optional, has defaults)
railway variables --set DB_POOL_MIN=2
railway variables --set DB_POOL_MAX=5
```

## 4. Initialize Database Schema

Link your project and run the schema:

```bash
# Link to your Railway project
railway link

# Run schema using Node.js script
railway run node scripts/run-schema.js
```

## 5. Deploy

Railway auto-deploys on git push. To manually deploy:

```bash
railway up
```

## 6. Update CORS After Deployment

After first deployment, update CORS with your actual domain:

```bash
railway variables --set CORS_ORIGIN=https://your-actual-domain.railway.app
```

## Notes

- `DATABASE_URL` is auto-generated by Railway when you add Postgres
- Generate encryption key: `node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"`
- Update Clerk webhook URLs to point to your Railway domain
- Railway provides separate URLs for each service (frontend/backend)
